An Initial Optimisation


Firstly give the current model a suitable name.
 

Edit -> Model -> Name
Change "Model_1(MgO.str)" to something suitable; say "MgO".


Note: For the change to take effect you must press Return. You'll see the name of the graphics window change to "DLV 3DView MgO";

Now open the Execute GULP panel.

Calculate -> Gulp -> Execute GULP

This panel will pop up;
 
 
 


 
 

As a first step use GULP to optimize the structure of MgO.

The symmetry of the crystal fixes the positions of the atoms within the cell and constrains the cell to be cubic. Thus there is only one structural parameter to optimize - the lattice constant a.

Select Optimization on the Execute GULP panel and press Run.

The Job List panel will open automatically;
 

 
 


 
 

Select the job and the status line should report "Job has completed" - like this;
 

 
 


 
 

Click on Recover Files and a window will open displaying the output from the GULP calculation (the LogFile). It should look something like  MgO_0.out .  Read through the LogFile.

Note that:

GULP describes MgO using the primitive unit cell. Initially the cell vectors are (in Angstrom);
 
        0.000000    2.105600    2.105600
        2.105600    0.000000    2.105600
        2.105600    2.105600    0.000000

This corresponds to a crystallagraphic (conventional) cell with lattice constant 2.9778 Angstrom.
You can see the two cells using Display->Structure. Turn on draw lattice and select either Primitive or Conventional as the cell type.


GULP reports the structure and the interaction potentials which describe the Mg-O bonding and then computes the energy for configuration 1 - the initial geometry. The inital energy is -40.99517 eV per unit cell. It then performs a bulk optimisation which reduces the energy to -40.99536 eV/cell with a final lattice constant of 2.9741 Angstrom. Clearly the initial geometry was very close to the minimum energy structure.
 
 
 
 
 
 

Select General Options and set the temperature to 300 Kelvin and the pressure to 0 Gpa.
